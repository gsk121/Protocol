# L2 데이터링크 계층 프로토콜

데이터링크 계층은 OSI 2계층으로, 같은 네트워크 세그먼트 내에서 노드 간 프레임 전송을 담당합니다.  
물리 주소(MAC 주소)를 사용하며, 오류 감지·수정 기능을 포함합니다.

---

## Ethernet II (IEEE 802.3)

| 항목 | 내용 |
|------|------|
| **구현 여부** | ✅ 구현됨 |
| **소스 파일** | `header/eth.py` |
| **표준** | IEEE 802.3 |

### 개요
이더넷은 현재 가장 널리 사용되는 유선 LAN 기술입니다.  
Ethernet II 프레임 형식은 목적지 MAC(6바이트) + 출발지 MAC(6바이트) + 타입(2바이트) + 페이로드로 구성됩니다.

### 프레임 구조
```
+------------------+------------------+-----------+------------------+
| Destination MAC  |   Source MAC     |  EtherType|    Payload       |
|    (6 bytes)     |   (6 bytes)      | (2 bytes) |  (46~1500 bytes) |
+------------------+------------------+-----------+------------------+
```

### 주요 EtherType 값
| 값 | 프로토콜 |
|----|---------|
| `0x0800` | IPv4 |
| `0x0806` | ARP |
| `0x86DD` | IPv6 |
| `0x8100` | IEEE 802.1Q (VLAN) |

### 구현 내용
- `struct.pack`을 사용한 이더넷 헤더 직렬화
- `!6s6sH` 포맷으로 목적지/출발지 MAC 및 EtherType 패킹

## ARP (Address Resolution Protocol)

| 항목 | 내용 |
|------|------|
| **구현 여부** | ✅ 구현됨 |
| **소스 파일** | `header/arp.py`, `posion1.py`, `posion2.py` |
| **표준** | RFC 826 |

### 개요
ARP는 네트워크 계층의 IP 주소를 데이터링크 계층의 MAC 주소로 변환하는 프로토콜입니다.  
브로드캐스트로 "이 IP를 가진 호스트는 MAC을 알려주세요"라는 요청을 보내고, 해당 호스트가 유니캐스트로 응답합니다.

### 패킷 구조
```
+--------+--------+--------+--------+----+----+------------------+------------------+------------------+------------------+
| HType  | PType  | HLen   | PLen   | Op |    | Sender MAC (6B)  | Sender IP (4B)   | Target MAC (6B)  | Target IP (4B)   |
| (2B)   | (2B)   | (1B)   | (1B)   |(2B)|    |                  |                  |                  |                  |
+--------+--------+--------+--------+----+----+------------------+------------------+------------------+------------------+
```

### 주요 Operation 코드
| 값 | 의미 |
|----|------|
| `1` | ARP Request |
| `2` | ARP Reply |

### 구현 내용
- ARP Request/Reply 헤더 생성 (`header/arp.py`)
- ARP Poisoning 공격용 가짜 ARP Reply 생성 (`posion1.py`, `posion2.py`)

### 보안 취약점: ARP Poisoning
ARP는 인증 없이 Reply를 신뢰하기 때문에, 공격자가 가짜 ARP Reply를 보내  
다른 호스트의 IP-MAC 매핑을 오염시킬 수 있습니다 (ARP Cache Poisoning).

## IEEE 802.1Q (VLAN Tagging)

| 항목 | 내용 |
|------|------|
| **구현 여부** | ❌ 미구현 |
| **소스 파일** | - |
| **표준** | IEEE 802.1Q |

### 개요
VLAN(Virtual LAN) 태깅은 하나의 물리 네트워크를 논리적으로 분리된 여러 네트워크로 나누는 기술입니다.  
Ethernet 프레임에 4바이트 태그(TPID + TCI)를 삽입하여 VLAN ID를 구분합니다.

### 태그 구조
```
+----------+----------+----------+----------+
|   TPID   |   PCP    |   DEI    |  VLAN ID |
| (2 bytes)|  (3 bits)| (1 bit)  | (12 bits)|
+----------+----------+----------+----------+
```
- **TPID**: `0x8100` (VLAN 태그 식별자)
- **PCP**: Priority Code Point (QoS 우선순위, 0~7)
- **DEI**: Drop Eligible Indicator
- **VLAN ID**: 1~4094 범위의 VLAN 번호

### 구현 계획
- VLAN 태그가 포함된 Ethernet 프레임 생성
- VLAN 태그 파싱 및 VLAN ID 추출
- 트렁크 포트 시뮬레이션

## PPP (Point-to-Point Protocol)

| 항목 | 내용 |
|------|------|
| **구현 여부** | ❌ 미구현 |
| **소스 파일** | - |
| **표준** | RFC 1661 |

### 개요
PPP는 두 노드 간의 직접 연결에서 사용되는 데이터링크 프로토콜입니다.  
주로 DSL, 다이얼업, VPN 터널링(PPPoE, PPTP)에서 사용되며 인증(PAP, CHAP), 압축, 암호화 기능을 지원합니다.

### 프레임 구조
```
+-------+----------+-------+----------+------------------+------+
| Flag  | Address  |Control| Protocol |     Data         | FCS  |
| (1B)  |  (1B)    | (1B)  | (2B)     |  (variable)      | (2B) |
+-------+----------+-------+----------+------------------+------+
```

### 구현 계획
- PPP 프레임 생성 및 파싱
- LCP(Link Control Protocol) 협상 시뮬레이션
- PAP/CHAP 인증 패킷 생성

## STP (Spanning Tree Protocol)

| 항목 | 내용 |
|------|------|
| **구현 여부** | ❌ 미구현 |
| **소스 파일** | - |
| **표준** | IEEE 802.1D |

### 개요
STP는 스위치 네트워크에서 루프를 방지하는 프로토콜입니다.  
BPDU(Bridge Protocol Data Unit) 메시지를 교환하여 루트 브리지를 선출하고, 중복 경로를 차단합니다.

### BPDU 구조
```
+----------+----------+----------+----------+----------+
| Protocol | Version  |  Type    |  Flags   | Root ID  |
| (2B)     | (1B)     | (1B)     | (1B)     | (8B)     |
+----------+----------+----------+----------+----------+
| Root Cost| Bridge ID| Port ID  | Msg Age  | Max Age  | Hello   | Fwd Dly |
| (4B)     | (8B)     | (2B)     | (2B)     | (2B)     | (2B)    | (2B)    |
+----------+----------+----------+----------+----------+---------+---------+
```

### 구현 계획
- BPDU 패킷 생성 및 파싱
- 루트 브리지 선출 로직 시뮬레이션
